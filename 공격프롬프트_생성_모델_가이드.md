# 공격 프롬프트 생성 모델 가이드

이 문서는 공격 패턴 초안 생성에 쓸 로컬 모델을 고르는 기준만 정리한다. 목적은 “모델 비교”가 아니라 “어떤 상황에서 어떤 모델을 쓰는지”를 빠르게 결정하는 것이다.

## 목적

- `data/attack_patterns/`에 넣을 초안 생성
- seed 다양성 확보
- 로컬 환경에서 반복 생성 가능

## 선택 기준

1. 거부율이 낮아야 한다.
2. 8GB급 VRAM에서도 돌릴 수 있어야 한다.
3. GGUF 또는 Ollama 등록이 쉬워야 한다.
4. 공격 패턴 다양성이 나와야 한다.

## 권장 사용 분류

### 1순위: 균형형

- Qwen 계열 7B~8B uncensored/abliterated
- 용도: 일반 공격 초안 대량 생성

### 2순위: 속도형

- 4B급 abliterated 모델
- 용도: 빠른 샘플 다량 생성

### 3순위: 사고형

- reasoning/distill 계열 8B
- 용도: 다단계, 우회 논리가 필요한 초안

## 사용 원칙

- 한 번에 한 카테고리만 생성한다.
- `technique`를 명시해서 다양성을 강제한다.
- 시스템 프롬프트에는 red-team 목적을 명확히 준다.
- 결과는 바로 실험에 쓰지 말고 검수 후 seed화한다.

## 출력 검수 기준

- 실제 요청처럼 자연스러운가
- injection 메커니즘이 분명한가
- 기존 seed와 중복되지 않는가
- 목표 카테고리와 맞는가

## 운영 팁

- Q4 계열부터 시작한다.
- 온도는 너무 낮추지 않는다.
- 생성 모델과 타겟 모델을 혼동하지 않는다.
- 분석 문서는 공식 기준 문서와 분리한다.
