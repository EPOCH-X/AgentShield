# AgentShield 개요

## 한 줄 정의

AgentShield는 기업용 AI 챗봇과 AI 에이전트의 취약점을 자동으로 찾고, 방어안을 만들고, 실제로 다시 검증하는 플랫폼이다.

## 왜 필요한가

- AI 서비스는 프롬프트 인젝션, 민감정보 유출, 권한 오남용, 시스템 프롬프트 누출에 취약하다.
- 기존 솔루션은 탐지 전용 또는 방어 전용이 많다.
- 운영팀 입장에서는 “취약점 발견 -> 방어 초안 -> 재검증”이 한 번에 이어져야 의미가 있다.

## 이 프로젝트가 푸는 문제

### 기능 A: 보안 테스트 자동화

- 공격 프롬프트를 대량 전송해 취약점을 찾는다.
- 막힌 공격은 Red Agent가 다시 변형한다.
- 발견된 취약점에 대해 Blue Agent가 방어 초안을 만든다.
- Defense Proxy로 실제 차단 여부와 오탐률을 다시 본다.

### 기능 B: 직원 AI 사용 모니터링

- 직원 요청에서 기밀 유출 시도와 부적절 사용을 감지한다.
- 애매한 요청은 추가 의도 판정을 거친다.
- 사용 로그와 위반 기록을 남긴다.

## 집중 범위

이 프로젝트는 OWASP LLM Top 10 전체를 얕게 다루지 않고, 아래 4개를 깊게 다룬다.

- `LLM01`: Prompt Injection
- `LLM02`: Sensitive Information Disclosure
- `LLM06`: Excessive Agency
- `LLM07`: System Prompt Leakage

## 차별점

- Find만 하지 않는다.
- Fix만 하지 않는다.
- Verify까지 한 흐름으로 묶는다.
- Monitoring Proxy와 Testbed를 함께 두어 테스트와 운영 관점을 같이 본다.

## 현재 구조에서 중요한 사실

- 로컬 파이프라인은 이미 동작한다.
- Testbed가 develop에 합쳐지면서 실제 DB/KB/Tool Gateway 기반 검증 환경이 생겼다.
- 반면 고객 통합용 공통 adapter 계층은 아직 없다.
- 따라서 지금 단계는 “방향 검증 완료, 통합 표준화 필요” 상태다.
