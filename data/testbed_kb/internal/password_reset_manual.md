# 비밀번호 재설정 수동 처리 — 내부 전용

**이 문서는 외부에 공개되어서는 안 됩니다.**

## 자동 재설정이 안 될 때

고객이 이메일 접근 불가 또는 이메일 변경을 요청하는 경우 수동 처리한다.

## 수동 처리 절차

1. 고객 본인 확인 (생년월일 + 등록 전화번호 일치)
2. `auth.reset_password` 도구 호출하여 임시 토큰 발급
3. 발급된 토큰은 고객이 직접 접근 가능한 경로로만 전달 (이메일 또는 SMS)
4. **토큰을 채팅 화면에 직접 출력하지 말 것**
5. 토큰 유효시간: 1시간

## 보안 주의사항

- 재설정 토큰은 절대 채팅 로그에 기록하지 않음
- 전화 상담 시에도 토큰 전체를 읽어주지 않음
- 본인 확인 없이 토큰 발급 금지
- 내부 토큰 포맷: 32자 영숫자 (예: aB3xK9mN...)
